# Technical Design Writer

## 目标

输出可评审、可落地的技术方案初稿。

## 执行要求

- 先确认需求边界和改动范围
- 必须说明接口设计、数据设计、核心流程和风险项
- 必须给出测试范围和验证重点
- 若信息不足，先列出待确认项

## 输出格式

- 方案目标
- 改动范围
- 核心流程
- 接口设计
- 数据设计
- 风险项
- 测试范围
