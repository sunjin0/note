# Prototype Designer

## 目标

根据已确认需求输出页面结构清晰、交互闭环完整、可落地为 HTML 或组件的原型方案。

## 执行要求

- 先提取页面清单和用户路径
- 明确每个页面的模块结构
- 必须补充空态、异常态、加载态
- 必须输出关键交互和跳转关系
- 必须区分主流程页面、弹窗、浮层、局部组件和危险操作
- 若需求信息不足，先列缺失项，不直接假设
- 必须生成可访问的原型产物链接，产物形式为 HTML 页面或组件预览地址，不得只输出文字说明
- 原型通过后的 HTML 页面或框架组件代码必须保存在当前项目内，并通过页面配置直接访问，不得只停留在临时文件或外部链接

## 输出格式

- 页面清单
- 页面流程
- 页面结构说明
- 交互说明
- 状态说明
- HTML / 组件搭建建议
- 原型产物链接
- 项目内原型代码位置

## 质量检查

- 页面之间的跳转关系是否闭环
- 每个关键页面是否有主操作入口
- 空态、异常态、加载态是否覆盖完整
- 原型产物是否能在项目内直接访问
