# 研发交付模板手册

> 用途：集中维护需求、方案、测试、发布、复盘、PR 描述、阶段检查和项目 SOP，避免模板文件分散。
> 配合文档：`docs/development-execution-workflow.md`

## 1. 使用原则

- 流程、门禁、职责、交付物口径以 `docs/development-execution-workflow.md` 为准。
- 本文档只负责提供统一模板、检查清单和执行顺序。
- 优先在项目或版本目录中复用本文档模板，不额外复制出多个结构相同的模板文件。

## 8. 阶段检查清单

- [ ] 业务目标明确

## 9. 项目交付 SOP

### 9.1 开工前

1. 明确版本目标和范围。
2. 指定产品、研发、测试负责人。
3. 建立需求文档和设计稿链接。
4. 自动生成项目检查清单。
5. 明确上线时间和版本边界。

### 9.2 需求澄清

1. 形成需求文档。
2. 自动生成验收标准和校验项。
3. 关闭或记录待确认项。
4. 完成可测性确认。

### 9.3 原型设计

1. 判断是否属于页面、交互或视觉承载类需求。
2. 输出页面清单、页面流和状态说明。
3. 生成可访问的 HTML 页面或组件原型。
4. 完成设计、产品、技术、测试联合确认。

### 9.4 方案设计

1. 输出技术方案。
2. 完成设计稿和关键流程说明。
3. 自动生成接口、数据、风险项检查清单。
4. 完成评审确认。

### 9.5 任务拆解

1. 形成开发任务和测试任务。
2. 自动生成联调点、依赖关系和排期清单。
3. 为每项任务指定责任人。

### 9.6 开发实现

1. 完成代码实现。
2. 自动执行静态检查和自测清单。
3. 完成自测。
4. 提交 PR，并关联需求、方案、设计稿。
5. 同步变更说明。

### 9.7 评审测试

1. 完成代码评审。
2. 执行功能测试和自动化回归。
3. 关闭阻塞问题或形成处理结论。
4. 输出测试结论。

### 9.8 发布上线

1. 确认发布范围与测试版本一致。
2. 自动生成发布验证清单。
3. 确认配置和回滚方案。
4. 执行发布。
5. 完成上线验证。

### 9.9 监控复盘

1. 收集监控、日志、反馈。
2. 自动生成复盘初稿。
3. 形成复盘记录。
4. 输出改进项并指定责任人。
