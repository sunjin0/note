## 2. 页面流程

### 2.1 主流程

应用锁屏页 -> 仪表盘页 -> 通过侧边栏切换到日历页 / 日记列表页 / 设置页

### 2.2 记录流程

仪表盘页 / 日历页 / 日记列表页 / 智能提醒 -> 心情编辑器弹窗 -> 保存 -> 刷新当前视图

### 2.3 模板流程

心情编辑器弹窗 -> 模板选择弹窗 -> 选择模板 -> 回填到编辑器 -> 保存

### 2.4 设置流程

设置页 -> 语言切换 / 因素管理 / 数据导出 / 清空数据 / 密码设置 / 安全问题找回

### 2.5 安全流程

应用启动 -> 检查会话状态 -> 未解锁显示锁屏页 -> 验证成功后进入主应用

## 3. 页面结构说明

### 3.1 布局说明

Web 版采用侧边栏 + 主内容区布局，重点适配桌面端宽屏操作。

### 3.2 应用锁屏页

用于密码保护启用后的进入门禁。

结构：

1. 应用标题与品牌说明
2. 密码输入框
3. 错误提示区
4. 解锁按钮
5. 忘记密码入口
6. 安全问题找回入口

### 3.3 仪表盘页

默认首页，承载快速记录入口和概览数据。

结构：

1. 顶部欢迎区
2. 今日心情快捷入口
3. 统计卡片区
4. 本周趋势图
5. 最近记录列表
6. 快速新建按钮

统计卡片建议包含：

- 连续记录天数
- 总记录数
- 最常见心情
- 积极心情比例

### 3.4 日历页

用于按日期查看和定位记录。

结构：

1. 月份切换区
2. 今天按钮
3. 月历网格
4. 当月统计摘要
5. 最近 90 天热图

关键说明：

- 日期格子需要显示是否有记录和对应心情颜色。
- 点击某日直接打开该日期的编辑器。

### 3.5 日记列表页

用于集中浏览、搜索和管理历史记录。

结构：

1. 搜索框
2. 高级筛选区
3. 排序与分页区
4. 分组列表区
5. 展开后的详情区

高级筛选建议包含：

- 日期范围
- 心情多选
- 因素多选

分组建议包含：

- 最近一周
- 按月归档
- 按年归档

### 3.6 设置页

用于配置语言、因素、安全和数据管理。

结构：

### 3.7 心情编辑器弹窗

## 4. 交互说明

### 4.1 全局交互

1. 侧边栏用于切换仪表盘、日历、日记列表和设置。
2. 移动端侧边栏以抽屉形式展示，点击遮罩层关闭。
3. 所有主视图切换后保持当前应用状态，不强制清空编辑上下文。
4. 语言切换后页面文案即时刷新。

### 4.2 记录交互

1. 点击“新建记录”或日期格子进入编辑器。
2. 编辑器中可先选心情，再补充文本、因素和照片。
3. 点击模板入口后打开模板选择弹窗，选中内容自动回填。
4. 保存成功后关闭编辑器，并刷新当前视图的统计和列表。
5. 编辑已有记录时，页面应默认带出当天已有数据。

### 4.3 日历交互

1. 点击上一月、下一月或今天后刷新对应月份内容。
2. 日期有记录时显示心情色块或标记。
3. 点击日期直接打开该日期的编辑器。

### 4.4 日记列表交互

1. 搜索输入后实时过滤列表，支持内容、因素、心情和日期。
2. 高级筛选变更后立即更新结果。
3. 点击卡片展开完整内容与照片。
4. 编辑按钮打开对应日期的编辑器。
5. 删除操作必须二次确认。

### 4.5 设置交互

1. 语言切换后立即生效。
2. 添加、编辑、删除自定义因素时均需保存后刷新列表。
3. 导出数据时显示导出成功或失败提示。
4. 清空数据必须二次确认，并明确不可恢复。
5. 密码设置与修改完成后应更新会话状态。

### 4.6 安全交互

1. 未通过验证时仅展示锁屏页。
2. 密码错误时显示错误提示和剩余次数。
3. 锁定期内禁止重复提交。
4. 忘记密码时走安全问题找回流程。

## 5. 状态说明

### 5.1 空态

1. 无记录时，仪表盘显示“开始记录你的第一条心情”。
2. 日历页无当月记录时，提示用户创建新记录。
3. 列表页无搜索结果时，提示“未找到符合条件的记录”。

### 5.2 加载态

1. 应用初始化时展示全局加载态或骨架屏。
2. 编辑器保存中禁用保存按钮并显示加载状态。
3. 数据导出和清空数据时展示处理中状态。

### 5.3 异常态

1. 密码错误、锁定和找回失败都需有明确错误反馈。
2. 照片上传失败、文件过大或数量超限时提示原因。
3. 本地存储异常时提示用户导出或清理数据。
4. 模板变量替换失败时提示异常并保留原文本。

### 5.4 编辑态

1. 当天已有记录时，编辑器进入回填状态。
2. 心情、因素、照片和日记内容都应支持回显。
3. 保存成功后，编辑态退出并同步刷新当前数据。

## 6. HTML / 组件搭建建议

1. 建议在同一份文档中建立一个 `Web` Frame 分组。
2. 优先搭建 6 个主页面：锁屏页、仪表盘、日历页、日记列表页、设置页、编辑器弹窗。
3. 弹窗类组件建议独立为可复用组件库。
4. 记录编辑器建议拆成顶部信息区、心情区、因素区、编辑区、图片区和底部操作区。
5. 日历页建议把“月历网格”和“90 天热图”分成两个区块，避免信息过密。
6. 日记列表页建议使用左侧筛选、右侧列表布局。
7. 所有页面需补齐空态、加载态、错误态和确认态。
8. 原型命名建议与代码模块保持一致，便于设计、开发、测试对齐，例如 `Dashboard`、`CalendarView`、`JournalList`、`Settings`、`MoodEditor`。

## 7. 已确认方案

1. 仪表盘统计卡片顺序固定为：`连续记录天数 -> 总记录数 -> 最常见心情 -> 积极心情比例`，采用“行为结果优先”的信息排序。
2. 日历热图与月历默认同屏展示，月历在上、热图在下，先定位日期再看趋势。
3. 富文本工具栏保留 `加粗 / 斜体 / 下划线 / 删除线 / 对齐 / 分隔线 / 撤销 / 重做 / 清除格式`，暂不扩展更多复杂能力。
4. 锁屏页保留“记住会话”入口，但默认关闭，以保证隐私保护强度。
